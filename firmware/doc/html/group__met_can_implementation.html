◆ MET_Can_Protocol_Loop()

void MET_Can_Protocol_Loop ( void  )

Application Main Loop function handler.

The function checks the flags rxReceptionTrigger or rxErrorTrigger.

In case of reception the function checks the CRC, data Lenght in order to proceed with the protocol decoding.

With the correct frame checked, the protocol is identified:

  • The Status Register is handled;
  • The Parameter Register is handled;
  • The Data Register is handled;
  • The Command frame is Handled;

The function activate the proper TX frame based on the received processed frame.

◆ MET_Can_Protocol_Reception_Callback()

void MET_Can_Protocol_Reception_Callback ( uintptr_t  context)
static

Interrupt routine.

CAN Reception Interrupt Handler.

This function is assigned in the MET_Can_Protocol_Reception_Trigger() function when the reception is activated. When the interrupt is executed it needs to be rescheduled in order to be executed again.

The function determines if the interrupt has been generated because of an error condition or because of a correct frame received. In both cases a given flag is set so it can be handled into the MET_Can_Protocol_Loop() function out of the Interrupt context.

Parameters
context

◆ MET_Can_Protocol_Reception_Trigger()

void MET_Can_Protocol_Reception_Trigger ( void  )
static

Reception activation routine.

This function triggers the reception of a further CAN frame.

The Function firstly rearm the interrupt handler to be launched. After the interrupt is armed, the function assignes the data pointer to the receving buffer.

In case of an error in activating the Reception, the error MET_CAN_PROTOCOL_ERROR_RECEPTION_ACTIVATION is signaled to the Error Handler routine.
