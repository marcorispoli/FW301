Detailed Description

GENERAL OVERVIEW

This protocol implements the functionalities of the PCB/22-301 Software Communication protocol specifications.

This module makes use of the application library module MET_Can_Protocol that implements the communication based mechanisms.

COMMUNICATION CHARACTERISTICS

The CAN communication is based on the following characteristics:

  • Standard (not Flexible) communication;
  • Standard frame (11 bit ID);
  • Baude Rate: 1Mb/s;
  • Can ID reception address: 0x201;

PROTOCOL DESCRIPTION

The Application iplements the protocol Items described in the PCB/22-301 Software Communication protocol specifications.

In this chapter it will be briefly described the protocol Items. For details refer to the document specification.

PROTOCOL STATUS REGISTERS

The available registers are:

  • RESERVED (IDX=0)
  • REVISION (IDX=1);
  • SYSTEM (IDX=2);
  • ERRORS (IDX=3);
  • COMMAND (IDX=4);
  • BATTERY (IDX=5);
